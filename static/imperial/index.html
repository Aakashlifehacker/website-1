Example Articles

* Here is a page with an article in a journal that Imperial do not appear to subscribe to, and that is not available anywhere (so it immediately offers to start an ILL):
http://stm.sciencemag.org/content/9/375/eaah6510

* Here is an article that is just generically open anyway, and (presumably) not in Spiral and not somewhere that Imperial subscribe to:
http://journals.plos.org/plosone/article?id=10.1371/journal.pone.0169629

* Here is an article in a journal that Imperial do subscribe to, although we do not find the journal itself (see PROBLEM, below), we do find that it is available to read locally:
https://www.sciandmed.com/sm/journalviewer.aspx?issue=1207&article=2077

The possible results the bookmarklet will currently return are:
* if an item is available in the local repo, show a blue "available" link to item
* else if relevant journal subscription appears to be available, show a blue link saying to sign in to gain access (PROBLEM - READ BELOW)
* else if available locally in library locations, show a blue "available locally" link, when clicked, lists the status and locations
* else if available anywhere else by the usual Open Access Button lookup, show blue "open article" link that opens the alternate source
* else if we find nothing, the button is red instead of blue and prompts to start an ILL
* a general error (if you see one, let us know how to reproduce it)

In each case above (except errors), there will also be a small link text offering to start an ILL if we appear to have the wrong item.

Exlibris Problem

Exlibris search for journals is not reliably returning the correct journal. For example, a search for the journal "science" from a page like http://science.sciencemag.org/content/293/5530/629 does NOT return the journal "Science" even though presumably Imperial do subscribe to it - because there are many other journals with the word "science" in their title, and starting with the word "science". Until I can find a reliable way to search exlibris for the actual journal title, we will have a problem here.

Major changes from mock-ups


* We've simplied some stages as the request system isn't intergrated here (and as we continue to refine concepts)
* We've allowed titles to be edited as there is no other point to correct these in the process currently. In our public button we have pages where these can be later checked and edited - we haven't yet shipped that for these yet. That can come after our sprint for the request system.
* Copy isn't refined yet.
* We're currently displaying where in the library physical items are, this probably shouldn't be included in future! For quick references, mockups and comments on them can be found here

Path fowards

The currently bookmarklet should demonstrate roughly what we're able to do although is clearly not a polished product. Thankfully, we were able to do everything we expected to some degree (bar the problme noted above). To take things forward we would:
* Refining copy and appearance
* Working on improving Exlibris search
* Implementing the request system into the tool (as discussion suggests we should)
* Removing the physical location box
* Other changes from your feedback
* Connect up the ILL request to Imperial's script & tweek format (currently just sends email)
